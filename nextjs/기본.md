# 사용자 인터페이스 렌더링
사용자가 웹 페이지를 방문하면, 서버는 HTML 파일을 브라우저에 반환한다.    
브라우저는 HTML을 읽고 DOM(Document Object Model, 문서 개체 모델)을 구성한다.
DOM 메서드와 프로그래밍 언어를 사용하여 사용자 인테페이스에서 특정 요소를 선택, 추가, 업데이트 및 삭제함으로써 사용자 이벤트를 수신하고 DOM 을 조작할 수 있다.     

> DOM이란? HTML 요소의 객체 표현이다. 코드와 사용자 인터페이스 사이의 다리 역할을 하여, 부모 및 자식 관계가 있는 트리와 같은 구조를 가진다.

> HTML? Hypertext Markup Language, 하이퍼 텍스트 마크업 언어이다. 웹 페이지가 어떻게 구조화 되어 있는지 브라우저로 하여금 알 수 있도록 하는 마크업 언어이다.      
>> elements 로 구성되어 있으며, 이들을 적절한 방법으로 나타내고 실행하기 위해, 각 컨텐츠의 여러 부분들을 감싸고 마크업 한다. `ex) <strong>이곳이 content 영역, 이 부분을 element라고 한</strong>`     
>> tags 는 웹 상의 다른 페이지로 이동하게 하는 하이퍼링크 내용들을 생성하거나, 단어를 강조하는 등의 역할을 한다. `ex) <strong></strong>`     


## DOM과 개발자가 만든 원본 HTML 파일과는 다르다
HTML 은 초기 페이지 콘텐츠를 나타내는 반면, DOM 은 작성한 자바스크립트 코드에 의해 변경된 업페이트된 페이지 콘텐츠를 나타낸다.    


# 명령형 프로그래밍과 선언형 프로그래밍
명령어 프로그래밍: 사용자 인터페이스를 업데이트 하는 방법에 대한 단계를 작성한다.   
선언형 프로그래밍: 단계를 추상화한다.   
개발 프로세스 속도를 높이기 위해, 선언적 접근 방식을 선호되는 경우가 많다.
